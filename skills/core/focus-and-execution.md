# Skill: Focus And Execution

Use this skill when the user:

- feels scattered
- starts many things and completes little
- loses momentum after early excitement
- struggles to follow through
- has limited time because of work or life constraints
- knows what to do in theory but does not do it in practice

## Goal

Help the agent turn intention into consistent execution without relying on hype, guilt, or unrealistic discipline fantasies.

## Core rule

Execution failure is not always a motivation problem.
It may be caused by:

- unclear priorities
- too much active scope
- plans bigger than the user's real capacity
- emotional avoidance
- attention fragmentation
- weak habit anchoring
- unrealistic time assumptions
- poor environmental setup

The agent should diagnose the execution failure before prescribing more effort.

## What to inspect

- what the user actually does in a normal week
- what the user says they will do versus what gets done
- where execution collapses
- how many active priorities exist
- whether the current plan fits real time and energy
- what distracts or derails follow-through
- whether the user needs a smaller loop, not a better speech

## Execution diagnosis

### 1. Clarity test

Does the user know the exact next step, or only the general goal?

### 2. Capacity test

Does the plan fit the user's real schedule, energy, and obligations?

### 3. Friction test

What makes starting hard?
Examples:

- too much ambiguity
- too much effort to begin
- emotional resistance
- switching cost
- missing tools or setup
- weak routine anchor

### 4. Scope test

Is the user trying to move too many priorities at once?

### 5. Behavior test

Is the problem:

- forgetting
- procrastinating
- avoiding
- overplanning
- self-deception
- mood-dependent effort

## Important distinctions

Separate between:

- inability to execute because the plan is bad
- inability to execute because the plan is too large
- inability to execute because the user is overloaded
- inability to execute because of behavioral avoidance

## Good intervention patterns

The agent may need to:

- shrink the plan
- reduce active priorities
- define one clear next action
- create a weekly execution rhythm
- anchor work to a specific recurring time
- remove unnecessary options
- ask for proof-based follow-up instead of vague updates
- force review before allowing new plans

## Anti-patterns

Avoid:

- giving long task lists to a scattered user
- assuming discipline will appear from motivation alone
- prescribing ideal routines that do not fit the user's life
- confusing temporary inspiration with execution readiness
- adding new goals when old ones are still unresolved

## Output requirements

When this skill is active, include:

- the main execution blocker
- what makes execution fail in this case
- the smallest workable execution loop
- how to fit it into the user's real week
- what should be removed or paused
- what proof of execution should appear next
