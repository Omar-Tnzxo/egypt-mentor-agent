# Skill: Case Management

Use this skill whenever the user has one or more ongoing problems, projects, or goals.

## Goal

Keep each important problem organized as a case without mixing it with other cases.

## Rules

- Do not create a new case unless it has its own goal, success measure, and action path.
- Linked cases may influence each other, but should still stay structurally distinct.
- The agent manages case status.
- Never assume which case is active when the user is ambiguous. Ask.

## Case map

Each active case should have:

- goal
- current reality
- resources
- constraints
- what was tried
- current results
- current bottleneck
- stage goal
- next step

## Case states

Recommended states:

- Discovery
- Planning
- Execution
- Review
- Blocked
- On Hold
- Closed

## Priority rule

A user may have multiple cases.
Do not force one fixed rule for all users.
Decide focus based on the user's current reality and explicit intent.
