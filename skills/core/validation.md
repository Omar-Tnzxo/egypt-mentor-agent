# Skill: Validation

Use this skill when the user has:

- an idea
- a startup
- a product with unclear traction
- users but no revenue
- activity without proof of real value

## Goal

Help the agent determine whether the case has real demand, real pain, and a real path worth pursuing before pushing growth or monetization.

## Core rule

Do not help the user scale confusion.
First test whether the product, offer, or direction is actually validated enough to deserve more effort.

## What to inspect

- who the real user is
- who the economic buyer is
- what painful problem is being solved
- how often the problem happens
- what the current workaround is
- whether users return because of real value or curiosity
- whether the product creates behavior change or only signups
- what evidence exists beyond opinions

## Validation layers

### 1. Problem validation

Check whether the problem is:

- real
- painful
- frequent
- important enough to solve now

### 2. User validation

Check whether the current users are:

- the right users
- active users
- repeated users
- valuable users

### 3. Value validation

Check whether the product creates a clear outcome:

- saved time
- better conversion
- easier workflow
- better visibility
- lower cost
- less friction

### 4. Buyer validation

Check whether the person using the product is the same person who can pay for it.
If not, identify the actual buyer and the actual economic reason to pay.

### 5. Monetization readiness

Do not jump to pricing until there is evidence that:

- the value is clear
- the user type is correct
- the buyer is identified
- the problem is strong enough

## Evidence priority

Prefer:

- active usage
- repeat usage
- user interviews
- retained behavior
- conversion behavior
- direct willingness to pay
- actual purchases

Over:

- vanity signups
- compliments
- social hype
- friend feedback
- downloads without retention

## If validation is weak

The agent should consider:

- narrowing the target user
- redefining the problem
- simplifying the offer
- talking to users before building more
- freezing growth work until clarity improves

## Output requirements

When this skill is active, include:

- current validation level
- what is validated
- what is not validated
- the biggest validation gap
- the next cheapest test that can reduce uncertainty
