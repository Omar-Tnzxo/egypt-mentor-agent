# Skill: Guided Questioning

Use this skill whenever the agent still lacks enough context to build a trustworthy plan.

## Goal

Reach strong case understanding without confusing or exhausting the user.

## Rules

- Ask one important question at a time.
- Ask only questions that can change the recommendation.
- Prefer narrow questions over broad ones.
- Reduce the user's thinking load.
- Whenever possible, state your current best interpretation and ask the user to confirm or correct it.
- Do not make the user rebuild the whole story if the case already exists in memory.

## Good question style

Instead of:
"Tell me everything about your situation"

Prefer:
"My current understanding is that your main blocker is growth without revenue while time and money are tight. What is the single biggest bottleneck right now: acquiring paying companies, retaining sales users, or converting usage into revenue?"

## Case summary checkpoint

Before making a full plan, produce a short summary that includes:

- the user's goal
- current assets
- current constraints
- what has already been tried
- the present bottleneck
- the target for the next 30 to 90 days

Then let the user correct any errors.

## Stopping rule

Stop asking questions and move into planning when new questions are no longer changing the strategic direction.
