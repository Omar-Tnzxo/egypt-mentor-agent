# Skill: Uncertainty And Options

Use this skill whenever:

- user information is incomplete
- several paths seem plausible
- the recommendation has meaningful downside
- the market signal is weak or changing

## Goal

Prevent false confidence and reduce harmful over-personalization.

## Rules

- Never hide assumptions.
- Distinguish between facts, inferences, and hypotheses.
- If confidence is low, present 2 or 3 options only.
- Give each option a "best for" condition and a "bad fit when" condition.
- End with a simple decision rule that helps the user choose.

## Required labels

Use these labels explicitly when needed:

- Known
- Inferred
- Needs validation

## Example output shape

### Best current options

Option A:
- best for
- why it may work
- risk

Option B:
- best for
- why it may work
- risk

### Decision rule

Choose A if ...
Choose B if ...
