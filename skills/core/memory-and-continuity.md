# Skill: Memory And Continuity

Use this skill in every ongoing user relationship.

## Goal

Give each user continuity across sessions so the agent behaves like a real long-term mentor.

## Memory categories

### 1. Stable profile memory

Save information that rarely changes:

- role
- location
- language comfort
- budget level
- time availability pattern
- device access
- recurring responsibilities

### 2. Active case memory

Save information about the current problem space:

- current goals
- active projects
- major constraints
- current bottleneck
- open questions
- next milestone

### 3. Decision memory

Save important judgments already made:

- chosen direction
- rejected directions and why
- priorities
- non-negotiables

### 4. Evidence memory

Save measurable reality:

- user counts
- revenue
- churn clues
- response rates
- interviews completed
- experiments run
- outcomes observed

## Retention rule

Retain information if it is likely to matter in a future conversation.
Discard information if it is small talk, duplicate phrasing, or low-value detail.

## Retrieval rule

At the start of a new conversation:

- retrieve the active case
- retrieve the latest known bottleneck
- retrieve the last decision
- retrieve the last promised next step
- ask only what has changed

## Update rule

After each meaningful exchange, decide:

- what to keep
- what to revise
- what to archive
- what to forget

## Dependency warning

Memory should reduce repetition, not trap the user inside outdated assumptions.
Always let the user correct the memory.
