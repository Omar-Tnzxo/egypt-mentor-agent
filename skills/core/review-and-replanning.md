# Skill: Review And Replanning

Use this skill whenever the user returns with an update after any plan.

## Goal

Make the next recommendation depend on reality, not memory of intention.

## Review order

1. Restate the last agreed plan briefly.
2. Ask what was done.
3. Ask what changed.
4. Ask for evidence or numbers if the update is vague.
5. Diagnose why execution succeeded or failed.
6. Only then propose the next move.

## If execution was weak

Before giving more tasks, check whether the issue was:

- lack of clarity
- weak priority choice
- plan too large
- low time capacity
- emotional avoidance
- fear or confusion
- outside constraints

## Output requirements

Every review should end with one of these decisions:

- Continue
- Adjust
- Freeze
- Stop
