# Skill: Prioritization

Use this skill when:

- the user has too many possible actions
- the case contains several plausible bottlenecks
- the user is busy but unclear on the next move
- there are many good ideas but limited time, money, or focus
- the user keeps asking what to do first

## Goal

Help the agent identify the single most important current priority instead of letting the user spread effort across too many directions.

## Core rule

A good plan is not the plan that covers everything.
It is the plan that solves the most important current bottleneck with the least waste.

The default is one main priority.
Only allow tiny parallel tests when they are clearly needed to reduce uncertainty between close competing priorities.

## What to inspect

- what is blocking progress right now
- which bottleneck is upstream of other problems
- which action has the highest leverage
- which action is actually executable within the user's constraints
- what is urgent versus what is merely loud
- what will create clarity, traction, revenue, or learning fastest

## Priority tests

### 1. Bottleneck test

Ask:
Which issue, if solved now, would unlock the most progress?

### 2. Leverage test

Ask:
Which action changes the future decision tree the most?

### 3. Evidence test

Ask:
Do we have enough evidence that this really is the main priority?
If not, what is the smallest step to verify it?

### 4. Execution-fit test

Ask:
Can the user realistically execute this priority with current time, money, and energy?

### 5. Distraction test

Ask:
Is this priority truly important, or is it just emotionally attractive, fashionable, or easier to think about than the real issue?

## Parallel micro-test rule

Allow parallel micro-tests only when:

- there is genuine uncertainty between close priorities
- the tests are small and fast
- the tests are cheap
- the tests will not meaningfully fragment execution
- each test has a clear success signal
- the purpose is to resolve uncertainty, not to keep many active priorities alive

## Common prioritization mistakes

Avoid:

- optimizing secondary problems
- doing growth before validation
- doing branding before value clarity
- doing pricing before buyer clarity
- adding features before understanding behavior
- keeping many parallel priorities with weak execution capacity

## If priorities are still unclear

The agent should:

- reduce the number of active possibilities
- identify the highest-probability bottleneck
- use one short test to reduce uncertainty
- avoid big plans until the current bottleneck is clearer

## Output requirements

When this skill is active, include:

- the current top priority
- why it is first
- what is explicitly not the priority right now
- what evidence supports the choice
- whether any parallel micro-test is justified
- what would cause the priority to change
