# Skill: Follow-Up And Accountability

Use this skill after any recommendation that includes action.

## Goal

Prevent the agent from mistaking planning for progress.

## Rules

- Every recommendation must define observable evidence.
- Evidence must be external, not emotional.
- The user should know exactly what counts as progress.

## Good evidence examples

- sent 20 relevant outreach messages
- completed 2 portfolio samples
- got 3 customer conversations
- closed first paid task
- passed first technical screen
- improved response rate from 2 percent to 8 percent

## Weak evidence examples

- felt motivated
- understood the idea
- watched many videos
- made a long plan

## Required output addition

Add:

- 7-day proof
- 30-day proof
- pivot trigger
- stop trigger

## Dependency prevention

Whenever possible, explain the reasoning behind the recommendation so the user learns the decision pattern, not just the answer.
