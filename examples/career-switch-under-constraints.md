# Example: Career Switch Under Constraints

## Why this example matters

This tests whether the agent can help a user choose a realistic career move instead of a fantasy identity plan.

## Likely skills involved

- `skills/core/intake-and-diagnosis.md`
- `skills/core/prioritization.md`
- `skills/core/focus-and-execution.md`
- `skills/core/human-behavior.md`
- `skills/domains/career-and-work.md`

## Sample opening message

"I want to change my career into tech, but I work full-time, I have family responsibilities, and I keep starting courses without finishing them. I do not know what path is realistic for me."

## What a weak agent would do

- tell the user to learn programming generally
- give a six-month roadmap without checking time and energy
- recommend the most prestigious path instead of the best-fit path
- confuse motivation with employability

## What a strong agent should do first

- identify the user's real weekly time and mental energy
- determine whether the goal is first income, long-term switch, or prestige
- ask for current proof of ability, not just interests
- separate the human-side execution problem from the market-side path decision

## Example of a strong next question

"Before we choose a lane, I need one hard constraint from your real life: in a normal week, how many hours can you protect consistently for this without lying to yourself?"

## Example validated case summary

- Goal: move toward a tech-related income path
- Current reality: full-time job, family obligations, repeated course-starting without completion
- Constraints: low time, limited energy, weak execution consistency
- Current bottleneck: no realistic entry wedge and no sustained proof-building system
- Stage goal: choose one realistic lane with a proof plan that fits current life

## Example current priority

Choose the narrowest viable entry path that the user can actually build proof in under current constraints.

## Why this is the priority

The user does not need a broad identity shift first.
The user needs one realistic market-facing path and a proof loop that can survive a full-time life.

## What is not the priority now

- consuming more general motivation
- exploring many tech lanes in parallel
- planning for a dramatic full career reinvention immediately

## Example current-phase plan

- choose one lane only based on market fit and time tolerance
- define one proof asset for that lane: project, portfolio piece, case study, or mock work
- set a fixed weekly rhythm with the fewest sessions possible
- measure completion, not enthusiasm

## Review trigger

Review after the first proof asset is completed or after two weeks of missed execution.

## What the user should return with next

- chosen lane
- protected weekly time blocks
- proof asset selected
- progress on completion
- where execution broke if it broke
