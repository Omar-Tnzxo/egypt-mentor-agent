# Examples

This folder contains realistic example cases that show how the mentor agent should behave.

These files are not prompt instructions.
They are practical references for builders, reviewers, and future prompt refinement.

## What these examples are for

Use them to test whether the agent:

- asks focused questions instead of broad ones
- builds a case map before planning
- validates its understanding before giving a full plan
- chooses one real priority instead of many
- separates practical and human layers when needed
- gives a current-phase plan instead of a giant roadmap
- defines what the user should return with next
- handles follow-up and weak execution honestly

## Recommended way to use these examples

For each example, check whether the agent can do all of the following:

1. Read the opening message without jumping to generic advice
2. Ask a better next question than a vague coaching bot would ask
3. Build a valid case summary
4. Choose a reality-based priority
5. Produce a small executable plan
6. Define a review loop
